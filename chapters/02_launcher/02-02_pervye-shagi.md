## 2.2. Что лаунчер делает первым делом

Сразу после того, как Windows передал управление точке входа `1cestart.exe`, лаунчер выполняет несколько подготовительных действий — настолько обыденных, что их легко пропустить, но все они влияют на дальнейшее.

**Определяет свой собственный путь.** Лаунчер вызывает `GetModuleFileName` и узнаёт, откуда он сам был запущен — обычно `C:\Program Files\1cv8\common\1cestart.exe`. Это нужно для одной важной вещи: чтобы найти каталог `common`, рядом с которым лежат каталоги версий платформы (`8.3.22.1923\bin\`, `8.3.23.2137\bin\` и так далее). Лаунчер всегда исходит из того, что версии платформы лежат **на уровень выше своего собственного каталога**.

**Определяет каталоги настроек текущего пользователя.** Через стандартные API Windows (`SHGetFolderPath` с константами `CSIDL_APPDATA` и `CSIDL_COMMON_APPDATA`) лаунчер вычисляет:

- `%APPDATA%\1C\1CEStart\` — пользовательский каталог настроек, обычно `C:\Users\Ivanov\AppData\Roaming\1C\1CEStart\`
- `%ALLUSERSPROFILE%\1C\1CEStart\` — общесистемный каталог, обычно `C:\ProgramData\1C\1CEStart\`

Если пользовательский каталог ещё не существует (первый запуск 1С под этим Windows-пользователем), он создаётся пустым.

**Загружает локализационные ресурсы.** Лаунчер определяет язык интерфейса Windows и подгружает соответствующие строки для своих окон. Это объясняет, почему окно запуска 1С на русской Windows показывается на русском, а на английской — на английском, без всяких настроек.

**Проверяет аргументы командной строки.** Если ярлык запустил лаунчер с аргументами (как мы обсуждали в части I, например `ENTERPRISE /S"server\base"`), они разбираются заранее. Это влияет на дальнейшее поведение: если в аргументах есть всё необходимое для запуска конкретной базы, окно выбора может даже не показаться — лаунчер сразу перейдёт к запуску. Если аргументов нет — будет показано окно со списком баз.

Все эти шаги занимают миллисекунды, и пользователь их не замечает. С точки зрения пользователя «ничего не произошло», но на самом деле лаунчер уже знает, кто он, где он, под кем работает и куда идти за настройками.
