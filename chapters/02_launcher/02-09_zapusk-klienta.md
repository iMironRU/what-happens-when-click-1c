## 2.9. Формирование командной строки и запуск целевого клиента

Все решения приняты. Лаунчер знает:

- **Какой исполняемый файл запускать** — например, `C:\Program Files\1cv8\8.3.22.1923\bin\1cv8c.exe`.
- **В каком режиме** — `ENTERPRISE` или `CONFIG`.
- **К какой базе подключаться** — строка из поля `Connect` записи `.v8i`.
- **Какие дополнительные параметры передать** — имя пользователя (если есть `WA=1`, передаётся текущий пользователь Windows; если в `.v8i` есть `Usr`/`Pwd`, передаются они), параметры отладки, локализация, режим аутентификации, и так далее.

Из всего этого собирается командная строка примерно такого вида:

```
"C:\Program Files\1cv8\8.3.22.1923\bin\1cv8c.exe" ENTERPRISE /S"srv-1c.company.local\accounting_prod" /WA+ /N"Иванов"
```

Расшифровка ключей:

- **`ENTERPRISE`** — режим запуска (Предприятие). Альтернатива — `CONFIG` для Конфигуратора.
- **`/S"server\base"`** — серверная база с указанием сервера и имени базы. Для файловой было бы `/F"путь"`, для веб — `/WS"url"`.
- **`/WA+`** — использовать аутентификацию Windows (плюс — включить, минус — выключить).
- **`/N"имя"`** — имя пользователя в ИБ.

Полный набор ключей запуска довольно большой — там есть параметры для отладки (`/Debug`, `/DebuggerURL`), для управления локализацией (`/L ru`, `/VL ru_RU`), для технологического журнала (`/Out`), для автоматического выполнения сценариев (`/Execute`), и многое другое. Часть этих ключей можно задать прямо в `.v8i`, часть — только через командную строку. Полный список — в документации платформы.

Сформированная командная строка передаётся в `CreateProcess` — тот же системный вызов Windows, через который Windows раньше запустил сам `1cestart.exe`. Создаётся новый процесс — `1cv8c.exe` или `1cv8.exe`, — наследующий учётку текущего пользователя.

**Что происходит с лаунчером после этого.** Лаунчер обычно **завершает свою работу сразу после успешного запуска целевого клиента**. Он выполнил свою задачу — выбрать правильную программу с правильными аргументами — и больше не нужен. Окно лаунчера закрывается, процесс `1cestart.exe` исчезает из списка процессов. Дальше всё происходит уже в новом процессе клиента, который начинает свою собственную жизнь.

Бывают исключения: если лаунчер был запущен в особом режиме (например, для добавления базы из присланного `.v8i`), он может остаться в памяти до завершения этой операции. Но в стандартном сценарии «выбрал базу — нажал Предприятие» лаунчер завершается мгновенно.

> **Параллель.** Это классический паттерн `exec`-style лаунчера: программа запускает другую программу и сразу завершается, не оставаясь в памяти как «родитель». В Unix это буквально вызов `exec()`, который **заменяет** текущий процесс новым (а не создаёт ещё один). В Windows нет точного аналога `exec`, поэтому используется `CreateProcess` + завершение лаунчера, что даёт практически тот же эффект: после запуска целевого клиента лаунчера в памяти больше нет.
