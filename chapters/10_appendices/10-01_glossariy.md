## Приложения

### Приложение A. Глоссарий терминов

Краткий справочник терминов, использованных в статье. В первую очередь — для тех, кто пришёл из других стеков и встретил незнакомое слово.

**1С:Предприятие** — режим работы платформы, в котором конечный пользователь работает с прикладным решением: вводит документы, формирует отчёты, ведёт учёт. Запускается из лаунчера кнопкой «1С:Предприятие». Технически — `1cv8.exe ENTERPRISE` или `1cv8c.exe` для тонкого клиента.

**1Cv8.1CD** — основной файл данных файловой информационной базы. Собственный формат 1С с страничной структурой. Внутри — и пользовательские данные, и код конфигурации.

**1Cv8.1CL** — служебный блокировочный файл, лежащий рядом с `1Cv8.1CD`. Координирует доступ нескольких клиентов к файловой базе.

**`1cestart.exe`** — лаунчер платформы 1С. Маленькая программа, единственная задача которой — выбрать правильную версию платформы и тип клиента, и запустить их. Лежит в `C:\Program Files\1cv8\common\`.

**`1cestart.cfg`** — конфигурационный файл лаунчера. Существует в нескольких экземплярах с разными приоритетами (пользовательский, общесистемный, корпоративный).

**`1cv8.exe`** — толстый клиент платформы 1С. Универсальный клиент, поддерживающий все типы форм и Конфигуратор. Запускается с разными ключами командной строки (`ENTERPRISE`, `CONFIG`).

**`1cv8c.exe`** — тонкий клиент платформы 1С. Облегчённый клиент, работающий только с управляемыми формами. Не поддерживает Конфигуратор.

**Аутентификация ОС** — механизм входа в 1С под текущим пользователем операционной системы, без ввода имени и пароля. Использует стандартный SSPI Windows (Kerberos или NTLM).

**Базовая (комплектация)** — самая ограниченная и дешёвая комплектация типовой конфигурации. Только файловая база, один пользователь, нельзя править конфигурацию.

**Веб-клиент** — версия тонкого клиента, реализованная в браузере на JavaScript/HTML. Отдаётся веб-публикацией. Не требует установки на машине пользователя.

**Внешнее соединение** — механизм программного подключения к информационной базе из других приложений (через COM-коннектор `V83.COMConnector`). В этой статье подробно не разбирался.

**Журнал регистрации** — внутренний журнал событий конкретной информационной базы. Хранится в служебных таблицах ИБ. Видны входы пользователей, изменения объектов, ошибки.

**Информационная база (ИБ)** — конкретный развёрнутый экземпляр прикладного решения вместе с накопленными данными. См. часть 0.1.

**Клиент-серверная база** — информационная база, обслуживаемая сервером 1С (`ragent`/`rmngr`/`rphost`) и хранящая данные в одной из поддерживаемых СУБД. Альтернатива — файловая база.

**Кластер 1С** — группа процессов сервера 1С (`ragent` + `rmngr` + один или несколько `rphost`), обслуживающих информационные базы. Может работать на одном хосте или быть распределённым по нескольким.

**Конфигурация** — прикладное решение как таковое: модель данных + код + формы + роли. См. часть 0.1.

**Конфигуратор** — режим работы платформы для разработчика и администратора. Запускается из лаунчера кнопкой «Конфигуратор». Технически — `1cv8.exe CONFIG`. В тонком клиенте недоступен.

**КОРП (комплектация)** — расширенная комплектация типовой конфигурации с дополнительными возможностями для крупных предприятий.

**Лаунчер** — он же `1cestart.exe`. См. выше.

**Метаданные** — машинное описание структуры конфигурации: какие есть объекты, реквизиты, формы, роли. Хранятся в служебных таблицах ИБ. См. часть 6.1.

**Модуль** — единица кода на встроенном языке 1С. Бывают модули объектов, модули форм, общие модули, модуль сеанса, модуль управляемого приложения. Каждый модуль компилируется в свой контекст (серверный/клиентский).

**Модуль сеанса** — специальный серверный модуль конфигурации, в котором живёт обработчик `УстановкаПараметровСеанса`. Компилируется на сервере при старте сеанса.

**Модуль управляемого приложения** — специальный клиентский модуль конфигурации, в котором живут обработчики `ПередНачаломРаботыСистемы`, `ПриНачалеРаботыСистемы` и другие глобальные клиентские обработчики.

**Параметры сеанса** — типизированные именованные значения, привязанные к конкретному сеансу пользователя и используемые прикладным кодом и шаблонами RLS. См. часть 5.3.

**Подсистема** — группировка прикладных объектов конфигурации по «смысловому назначению». Из подсистем строится панель разделов главного окна. См. часть 8.3.

**ПРОФ (комплектация)** — стандартная коммерческая комплектация типовой конфигурации. Можно работать в клиент-серверной схеме, можно нескольким пользователям, можно дорабатывать конфигурацию.

**`ragent`** — агент сервера 1С. Служба, управляющая остальными процессами кластера на конкретном хосте. По умолчанию слушает порт 1540.

**Расширение конфигурации** — отдельный объект, который подключается к основной конфигурации в момент работы и меняет её поведение, не трогая исходников. См. части 0.7 и 6.2.

**Регистр накопления** — один из типов прикладных объектов конфигурации. Предназначен для накопления количественных или суммовых остатков по измерениям. Документы делают «движения» в регистрах.

**Регистр сведений** — другой тип прикладных объектов. Предназначен для хранения характеристик, которые могут меняться во времени (курсы валют, цены, статусы).

**Реквизит** — поле объекта конфигурации (справочника, документа). Аналог «колонки» в обычной БД, но с типизацией и метаданными.

**Роль** — именованный набор разрешений в конфигурации. Пользователю назначается одна или несколько ролей, и его итоговые права — объединение прав всех ролей.

**`rmngr`** — менеджер кластера 1С. «Мозг» кластера, который знает про базы, рабочие процессы, сеансы, балансировку. По умолчанию слушает порт 1541.

**`rphost`** — рабочий процесс сервера 1С. Процесс, в котором реально исполняется прикладной код, формируются ответы на запросы клиентов, происходит обмен с СУБД. По умолчанию слушает порты из диапазона 1560–1591.

**RLS (Row-Level Security)** — механизм ограничения доступа к данным на уровне отдельных строк. В 1С реализован через шаблоны ограничений в ролях. См. часть 6.3.

**Сеанс** — активная сессия работы пользователя с информационной базой. Имеет уникальный номер, видна в консоли администрирования, занимает одну клиентскую лицензию.

**Сервер 1С** — служба, обслуживающая клиент-серверные информационные базы. Состоит из процессов `ragent`/`rmngr`/`rphost`. Не путать с сервером СУБД — это две разные сущности.

**Серверная лицензия** — лицензия на сам сервер 1С. Нужна для запуска кластера и обслуживания клиент-серверных баз. Существует независимо от клиентских лицензий.

**Серверный вызов** — обращение клиентского кода к серверу для выполнения серверной процедуры. В управляемых формах серверные вызовы стоит минимизировать, потому что они идут через сеть.

**Справочник** — один из типов прикладных объектов. Предназначен для хранения списков объектов с иерархией: контрагенты, номенклатура, сотрудники, и так далее.

**`srvinfo`** — служебный каталог агента сервера 1С, в котором хранится «база данных» кластера: список зарегистрированных баз, параметры подключения к СУБД, журналы регистрации, контексты сеансов.

**Тонкий клиент** — он же `1cv8c.exe`. Облегчённый клиент платформы. Работает только с управляемыми формами. Может подключаться напрямую к серверу 1С или через веб-публикацию.

**Толстый клиент** — он же `1cv8.exe` в режиме `ENTERPRISE`. Полнофункциональный клиент. Поддерживает обычные и управляемые формы. Запускается также в режиме Конфигуратора.

**Управляемая форма** — современная модель форм в 1С, описанная декларативно в метаданных и рендерящаяся каждым клиентом по-своему (нативный WinAPI, Qt, HTML/JS). См. часть 8.5.

**`УстановкаПараметровСеанса`** — обработчик в модуле сеанса, который вычисляет значения параметров сеанса при первом обращении к ним. См. часть 5.3.

**Файловая база** — информационная база, хранящаяся в виде каталога с файлом `1Cv8.1CD`. Не требует сервера 1С. Подходит для одиночной работы или малых установок.

**HASP-ключ** — USB-ключ защиты от компании Thales (бывший SafeNet/Aladdin), используемый для аппаратной лицензии 1С.
