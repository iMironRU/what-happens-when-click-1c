## 8.7. Открытие рабочего стола и передача управления пользователю

Все шаги пройдены. Главное окно построено, обработчики начала работы отработали, начальная страница отрисована, формы готовы к открытию. Платформа делает последнее: **показывает пользователю главное окно и передаёт ему управление**.

С точки зрения процесса клиента это означает, что цикл обработки сообщений (классический Windows message loop в `1cv8.exe`/`1cv8c.exe`, или цикл событий браузера для веб-клиента) начинает обслуживать действия пользователя: клики мышью, нажатия клавиш, прокрутку колесом. Каждое действие пользователя превращается в вызов соответствующего обработчика — клиентского, серверного, или комбинации.

С этого момента сеанс **вошёл в нормальный рабочий режим**. Дальнейшая работа сеанса — это уже не «запуск», а «эксплуатация»: пользователь открывает формы, читает данные, вводит документы, формирует отчёты. Каждое действие проходит через ту же машинерию, которую мы разбирали: запросы транслируются в SQL и идут в СУБД через `rphost`, права проверяются по кешу прав, RLS добавляется автоматически, формы сериализуются и передаются клиенту, клиент рендерит их через свой бэкенд.

Сеанс будет жить до тех пор, пока:

- Пользователь сам не закроет клиент (нажав «Файл → Выход» или закрыв окно).
- Не истечёт время простоя сеанса (если оно настроено в свойствах базы).
- Администратор не завершит его принудительно из консоли кластера.
- Не упадёт `rphost`, обслуживающий сеанс (тогда сеанс может быть подхвачен другим `rphost` — или может умереть с потерей текущей операции, в зависимости от настроек кластера).
- Не порвётся сетевое соединение (с теми же возможными исходами).

Когда сеанс закрывается, платформа вызывает соответствующие обработчики завершения работы (`ПередЗавершениемРаботыСистемы`, `ПриЗавершенииРаботыСистемы`), освобождает занятую клиентскую лицензию, удаляет запись о сеансе из активного списка кластера (запись в журнале регистрации остаётся), и завершает контекст сеанса в `rphost`. Память освобождается, ресурсы возвращаются в общий пул.
